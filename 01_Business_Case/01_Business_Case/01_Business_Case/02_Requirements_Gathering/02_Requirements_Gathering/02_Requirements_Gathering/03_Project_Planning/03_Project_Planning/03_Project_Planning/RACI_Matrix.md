# RACI Matrix 🤝

## Purpose

This RACI matrix defines roles and responsibilities across the corporate travel platform implementation for ABC GmbH.

RACI helps create clear ownership, reduce confusion, and ensure that every major implementation activity has the right stakeholders involved.

## RACI Definitions

* **R — Responsible:** Person or team doing the work
* **A — Accountable:** Person who owns the final decision or outcome
* **C — Consulted:** Stakeholder providing input before decisions are made
* **I — Informed:** Stakeholder kept updated on progress or outcomes

## Key Roles

| Role                     | Description                                                                                                           |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------- |
| Implementation Manager   | Owns implementation planning, coordination, documentation, risk tracking, training preparation, and go-live readiness |
| Project Sponsor          | Senior client-side decision maker responsible for final approval and business alignment                               |
| HR Manager               | Provides employee data, user access requirements, and training audience information                                   |
| Finance Manager          | Owns travel policy, cost center, invoice, and reporting requirements                                                  |
| Operations Lead          | Supports approval workflow, internal process alignment, and rollout coordination                                      |
| IT Contact               | Reviews SSO, HR data import, and technical integration requirements                                                   |
| Product / Technical Team | Supports product configuration, technical feasibility, integrations, and issue resolution                             |
| Account Manager          | Takes over the client relationship after implementation and go-live                                                   |
| Department Managers      | Approve travel requests and support employee adoption                                                                 |
| Employees                | End users of the travel booking platform                                                                              |

## Ownership Notes

* The Implementation Manager drives structure, timelines, documentation, coordination, and follow-up.
* The Project Sponsor remains accountable for major client-side decisions.
* Finance owns travel policy, budget limits, and reporting requirements.
* HR owns employee data and user access inputs.
* IT and Product / Technical teams own technical feasibility and integration review.
* The Account Manager becomes accountable after handover and long-term relationship management.

## Implementation Value

Using a RACI matrix helps the project team avoid unclear ownership, delayed decisions, duplicate communication, and missed handover points during onboarding.
