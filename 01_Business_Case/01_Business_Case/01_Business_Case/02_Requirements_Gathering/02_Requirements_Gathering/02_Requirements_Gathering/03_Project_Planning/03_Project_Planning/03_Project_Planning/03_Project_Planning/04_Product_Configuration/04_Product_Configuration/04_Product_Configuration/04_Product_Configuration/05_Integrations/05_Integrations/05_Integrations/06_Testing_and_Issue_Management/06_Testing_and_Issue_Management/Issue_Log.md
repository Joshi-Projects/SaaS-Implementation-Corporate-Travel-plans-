# Issue Log 🛠️

## Purpose

This issue log tracks problems, blockers, and open items identified during testing and implementation.

The goal is to create clear ownership, priority, next steps, and follow-up so that critical issues are resolved before go-live.

## Priority Definitions

### High Priority

Issue may delay go-live, block core functionality, or affect user access, approval workflow, or reporting readiness.

### Medium Priority

Issue may create rework, confusion, or process gaps but does not fully block launch.

### Low Priority

Issue can be resolved before or shortly after go-live without major business impact.

## Issue Management Process

1. Identify the issue.
2. Log the issue with a clear description.
3. Assign an owner.
4. Set priority.
5. Define the next action.
6. Track status until resolution.
7. Retest if the issue affects configuration or workflow.
8. Include unresolved items in handover if not blocking go-live.

## Implementation Value

A structured issue log helps the Implementation Manager maintain visibility, reduce missed follow-ups, and ensure blockers are addressed before launch.
